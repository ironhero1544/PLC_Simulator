# 사용자 계획: PLC 컴파일 파이프라인 및 동기화 개선

- 작성일: 2025-08-13
- 작성자: 사용자
- 범위: Phase 1~5

## 개요
PLC 컴파일 파이프라인 안정화 및 I/O 동기화, 상태 가시성, 테스트 체계 구축을 5~6일 내에 단계적으로 수행한다.

## Phase 1: 컴파일 파이프라인 고정 (1-2일)
1) SetLadderProgram 수정 (ProgrammingMode.cpp:221)
- m_plcExecutor->LoadCompiledCode(ldContent) 제거
- dirty = true 플래그만 설정하도록 변경

2) 재컴파일 트리거 구현
- m_programHash 멤버 추가 (래더 구조 해시)
- m_isDirty 플래그 도입
- NeedsRecompilation() 로직 강화

3) ExecuteWithOpenPLCEngine 수정
- 시작 시 dirty/해시 체크 후 CompileLadderToOpenPLC() 호출
- 성공 시 m_currentCompiledCode 갱신, m_isDirty = false

## Phase 2: 입출력 동기화 단일화 (1일)
1) 입력 경로 통합
- UpdateInputsFromSystem(map<string,bool> inputs) 함수 추가
- 물리/IO 매핑에서 단일 진입점으로 X 상태 반영

2) 출력 경로 개선
- SyncOpenPLCToDevices() 범위 설정 (Y0~Y15, M0~M999)
- 경계 검증 및 상태 변화만 반영 최적화

## Phase 3: 타이머/카운터 초기화 (1일)
1) T/C 주소 스캔
- SetLadderProgram() 시 래더에서 T/C 주소 집합 생성
- 미존재 T/C 기본 상태로 초기화

2) preset 파싱 강화
- 정수 유효성 검증 및 경고 로그

## Phase 4: 상태 가시성 강화 (1일)
1) 엔진 상태 패널
- Engine 타입 (Compiled/Legacy)
- 컴파일 코드 로드 상태
- 마지막 컴파일/스캔 에러
- 성능 정보 (명령어 수, 사이클 시간)

2) 에러 메시지 개선
- 상태바/콘솔 요약 + 디버그 패널 상세 정보

## Phase 5: 테스트 및 검증 (1일)
1) 회귀 테스트 시나리오
- 기본 로직 (X0→XIC→OTE Y0)
- 수직 OR 연결
- TON/CTU 동작
- 편집→재컴파일 흐름
- 컴파일 실패 폴백

## 일정
- 총 예상 기간: 5-6일

## 리스크와 완화책
- 리스크: 대형 래더 컴파일 지연, SetLadderProgram 호출부 영향
- 완화책: dirty/해시 기반 지연 컴파일, 점진적 롤아웃
